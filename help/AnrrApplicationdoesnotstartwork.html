An rrApplication does not start/work: How to find the problem?

rrApplication log files

If the rrClient or rrServer have been started and they can write to the RR network folder, then they will write their log file into [RR]\sub\log\....

Even if they close at startup because some important function failed, they should start the log and write the error into that file.

If there is no log from the current time at all, then they have not been able to start or no write access.



rrAutostart Service log files

If you have installed the rrClient or rrServer as service/deamon, then please check the service installation directory.

By default the path is C:\Program files (x86)\rrService (windows) or /usr/local/rrService (Linux, OSX).

If the service has any issues starting the rrClient/rrServer, then there will be a file rrAutostartservice_Errors.log.


Note:

The file can be older, please check the time/date.


Debug service log:

You can edit the file rrAutostartservice.cfg and change

WriteLogFile = 1

If you now restart the service, it will print detailed information about what it does.

(Note: the service waits 30s before it does anything)


Remember to reset that change after you are done finding the problem.




Possible errors:

Error

Description

What to do?

Source file does not exists! \\.....\bin\win\rrClientcommander.exe

Logon failure: unknown user name or bad password

The service can not access the fileserver or the folder.

Please check the rrService user you have set for the rrService. The user is shown in rrAutostartservice.cfg as loginUser=.


Please confirm that this rrService user is known to your fileserver. If you want to change the user, please use rrWorkstationInstaller.


Please try to login on the machine with the rrService user. Windows can show an advanced error message why the user cannot login.

               


System Process List

Open the application that shows all processes on your OS. (Task Manager, System Monitor, Activity Monitor).

Check the option to show processes of all users (if it exists)


If the Client is started as service, then you should have these processes:

  • rrAutostartservice (the service/daemon)
  • rrClientconsole  (the client itself)
  • rrClientWatch  (optional, the UI for the client)



Manual Start

If there is an issue with the client not starting or not connecting, then try to start it manually.


First shut down the daemon/service if it is running. (via windows system/services or "/etc/init.d/rrAutostartservice stop" or just kill the process).

If the rrClientconsole process is running, kill the process.


Start rrClient or rrClientconsole.

Do you see any error messages? Can it connect?
